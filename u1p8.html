Paradigmas de programacion

Como en todo, siempre existe una alternativa para llegar a una solución, una vía alterna, una segunda opción, etc. De la misma manera, en la programación existen diferentes maneras o estilos de programar

Los paradigmas de desarrollo móvil son enfoques, metodologías o estilos que se utilizan para crear aplicaciones para dispositivos móviles

PWA
Son aplicaciones mejoradas con capacidades de instalacion y ejecucion sin conexion a internet asemejándose a las apps nativas

Tecnologias y herramientas
Service worker apis de web app manifest
Frameworks como angular react y vue

Ventajas
No requieren una instalacion desde una tienda
Funciona sin conexión
Se pueden actualizar facilmente
Desventajas
No tienen acceso completo a todas las funciones del dipositivo
Dependen del navegador


Hibridas
Permite crear una unica aplicación que se puede ejecutar en múltples plataformas, combinando tecnologías web con acceso a funciones nativas mediante herramientas de compilación

Tecnologías y herramientas
Frameworks como ionic apache cordova
Phone gap
Basado en html css javascript


Frameworks multi plataforma
Este paradigma permite escribir un solo código y compilarlo en codigo nativo para cada sistema operativo

Flutter
React native
Xamarin

Ventajas: codigo compartido entre plataformas rendimiento similar al nativo
Mayor rapidez de desarrollo

Desventajas
Puede haber problemas de compatibilidad con algunas funciones nativas
Mayor curva de aprendizaje en algunos frameworks


Nativas
Este paradigma se basa en el desarrollo de aplicaciones específicamente para un sistema operativo móvil como android o IOS utilizando lenguajes y herramientas oficiales de cada plataforma.

Tecnología y herramientas
Android java, kotlin, android studio
Ios: swift, objective c xcode

Ventajas:
Maximo rendimiento y eficiencia
Acceso completo a funcionalidades del hardware y software del dispositivo
Mejor ux

Desventajas:
Se requiere desarrollar por separado para cada plataforma mayor tiempo y costo de desarrollo
